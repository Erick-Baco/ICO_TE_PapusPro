Conceptos de Ingeniería de Software

¿Qué es?

Es el estudio sistematico, disciplinado y cuantificable de la aplicacion de un conjunto de metodologias tecnicas, procesos, herramientas, tecnicas de desarrollo, operacion y mantenimiento de software de alta calidad.

Objetivos

  • Calidad
  • Herramientas
  • Procesos
  • Metodos

Herramientas de Modelado UML

¿Qué es UML?

UML (Lenguaje Unificado de Modelado) es un estándar para visualizar, especificar, construir y documentar sistemas de software orientados a objetos.

Tipos de diagramas UML

  • Diagrama de clases
  • Diagrama de casos de uso
  • Diagrama de secuencia
  • Diagrama de estados
  • Diagrama de actividades

Modelo de Casos de Uso

¿Qué es?

Es una herramienta UML que describe las interacciones entre los actores (usuarios o sistemas externos) y el sistema, mostrando los servicios o funcionalidades que ofrece.

Elementos principales

  • Actores (usuarios, sistemas externos)
  • Casos de uso (funciones o servicios)
  • Relaciones (asociación, inclusión, extensión)

Diagrama de Clases

¿Qué es?

Representa la estructura estática del sistema, mostrando las clases, atributos, métodos y las relaciones entre ellas.

Aplicación

  • Diseño orientado a objetos
  • Visualización de dependencias y jerarquías
  • Identificación de patrones de diseño

Diagrama de casos de uso detallados

¿Qué es?

Muestra cómo interactúan los objetos entre sí en el tiempo, detallando el flujo de mensajes en un escenario específico.

Componentes clave

  • Objetos o instancias
  • Líneas de vida
  • Mensajes
  • Activaciones

Patrones de Diseño

¿Qué son?

Son soluciones reutilizables a problemas comunes en el diseño de software. No son código específico, sino plantillas para resolver estructuras frecuentes.

Tipos comunes

  • Creacionales (Factory, Singleton)
  • Estructurales (Adapter, Composite)
  • Comportamiento (Observer, Strategy)
  • Modelo Vista Controlador

Principios de Diseño SOLID

¿Qué es SOLID?

Un conjunto de cinco principios para escribir código mantenible y comprensible en programación orientada a objetos.

  • S: Responsabilidad única (Single Responsibility)
  • O: Abierto/Cerrado (Open/Closed)
  • L: Sustitución de Liskov
  • I: Segregación de interfaces
  • D: Inversión de dependencias

Ciclo de Vida del Desarrollo de Software

¿Qué es?

El ciclo de vida del software es el proceso completo de planificación, creación, prueba, despliegue y mantenimiento de un sistema de software.

Fases típicas

  1. Recolección de requerimientos
  2. Análisis
  3. Diseño
  4. Implementación
  5. Pruebas
  6. Despliegue
  7. Mantenimiento
