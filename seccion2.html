Patrones de diseño

Patron singleton

El patrón Singleton garantiza que una clase tenga una única instancia y proporciona un punto de acceso global a ella. Es útil para objetos que deben tener una sola representación, como una conexión a base de datos o un manejador de configuración.

  • Solo permite una instancia de la clase.
  • Control centralizado de acceso.
  • Se accede mediante un método estático (getInstance()).
  • Evita la creación de múltiples objetos innecesarios.

Patron Factory

El patrón Factory encapsula la lógica de creación de objetos, permitiendo que las subclases decidan qué clase instanciar. Es ideal cuando el código cliente no debe depender de las clases concretas que necesita utilizar.

  • Proporciona una interfaz para crear objetos, pero deja a las subclases la decisión de qué instancia crear.
  • Promueve la desacoplación entre el código cliente y las clases concretas.
  • Facilita la extensión del sistema sin modificar el código existente.
  • Útil para manejar familias de objetos relacionados.

Patron Adapter

El patrón Adapter actúa como un puente entre dos interfaces incompatibles. Permite que clases con interfaces distintas trabajen juntas envolviendo una en una interfaz compatible para el cliente.

  • Convierte la interfaz de una clase en otra esperada por el cliente.
  • Favorece la reutilización de clases existentes sin modificarlas.
  • Puede usarse para integrar sistemas heredados con nuevas funcionalidades.
  • Existen dos tipos: adapter de clase (herencia) y adapter de objeto (composición).
